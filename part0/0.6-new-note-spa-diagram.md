# 0.6: New note in Single page app diagram

Diagram depicting the situation where the user creates a new note using the single-page version of the app at https://studies.cs.helsinki.fi/exampleapp/spa.

```mermaid
sequenceDiagram
    participant browser
    participant server

    Note right of browser: User types a note and clicks "save"

    Note right of browser: The submit handler prevents the default form submission, adds the note to the local notes array, redraws the list on the page, and then sends the new note to the server

    browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/new_note_spa
    Note right of browser: Request body: { content: "new note content", date: "2026-04-22T..." } as JSON<br/>Header: Content-Type: application/json

    activate server
    Note left of server: Server parses the JSON body and pushes the new note into the notes array
    server-->>browser: HTTP 201 Created
    deactivate server

    Note right of browser: No page reload. No additional HTTP requests. The UI was already updated optimistically before the response arrived.
```
